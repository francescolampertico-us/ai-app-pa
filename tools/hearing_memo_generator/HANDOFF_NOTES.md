# Handoff Notes

## What changed

The package has been updated using the submitted final memo as the primary style reference.

This means the prototype should now aim for:
- house-style memo structure
- the exact default headings
- member-by-member Q&A
- high-level hearing overview
- no extra closing section by default
- a single inline confidentiality line, with repeated footers handled in export

## What to test first

Run the prototype on the same hearing used for the submitted memo and compare:
- heading strings
- speaker-label format
- level of abstraction in the overview
- paragraph rhythm
- how the chair closing is handled
- footer behavior

## What to ignore in first-pass testing

Do not over-index on perfect docx formatting before the text output is correct.
Get the memo logic right first, then tighten rendering.
