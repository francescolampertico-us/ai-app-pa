# Example 1: Expected Output Structure

The generator should produce:

## Message House
- **Core Message**: A single quotable sentence capturing the position
- **Pillar 1** (e.g., "Consumer Protection"): Argument grounded in safety testing provisions + proof points from the bill context
- **Pillar 2** (e.g., "Innovation Leadership"): Argument about maintaining US competitiveness + liability safe harbor proof point
- **Pillar 3** (e.g., "Bipartisan Governance"): Argument about responsible oversight + 19-7 committee vote proof point
- **Key Terms**: Specific phrases like "pre-deployment safety testing", "frontier AI", "responsible innovation"
- **Target Audiences**: Senate Commerce Committee, tech industry, media

## Deliverables
Each variant should:
1. Be consistent with the core message and pillars
2. Reference specific provisions from the bill context (compute threshold, NIST, $500M)
3. Address the target audience (Senate Commerce Committee)
4. Follow the format constraints (word count, tone, structure)
5. Not fabricate statistics or provisions not in the context
