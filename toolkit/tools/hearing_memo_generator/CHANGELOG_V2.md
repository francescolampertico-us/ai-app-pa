# Changelog - Package Update from Submitted Memo

This update uses the submitted final memo as the primary style reference.

## Main changes

- locked the default memo family to the house-style structure
- made the overview explicitly high level and non-speaker-specific
- made Q&A organization member-by-member rather than issue-cluster-first
- made chair closing remarks part of the chair subsection by default
- tightened speaker-label rules for headings vs body text
- set the default confidentiality line to:
  - Confidential - Not for Public Consumption or Distribution
- clarified that repeated page footers belong to export formatting, not text generation
- upgraded verification to check day/date conflicts and section-placement errors

## Why this matters

The previous package captured the broad architecture but not enough of the actual memo-format logic.
This update should make prototype outputs much closer to the submitted memo family.
