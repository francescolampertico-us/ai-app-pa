## Exact Repro Steps

2. Start from the `MLB-SMK-01` input in `toolkit/qa/test_cases/media_list_builder.md`.
3. Review the location controls while switching between `National (US)`, `State`, and `City / Metro`.
4. Observe how the secondary location input behaves.

## Exact Pasted Input
- Policy issue: `AI safety regulation and mandatory pre-deployment testing requirements`
- Geographic scope: `National (US)` initially
- Number of contacts: `20`
- Media types: `mainstream, print, broadcast, digital, trade`

## Observed Behavior
The location UI is confusing/inconsistent around national vs city/metro selection. The current control design makes the geographic scope harder to understand than it should be and does not clearly communicate when city/metro selection is available or in effect.

## Expected Behavior
The geographic scope UI should make National, State, and City / Metro behavior explicit and intuitive. Secondary location input should appear, hide, enable, or disable in a way that clearly matches the chosen scope.

## Why This Matters
This is a core input control for the tool. If the user cannot clearly understand the scope they are sending, they cannot trust the resulting media list.

## Evidence / Output Excerpt
User report:

```text
when selected national, cannot pick the city metro
```

Code context:

```text
The page uses one shared location input that is disabled for `National (US)` and relabeled between `State` and `City / Metro`, which creates an unclear interaction model.
```

## Suspected Layer
`frontend`

## Claude Fix Request
Improve the Media List Builder geographic scope controls so National, State, and City / Metro are clear and consistent. Make the UI behavior match the selected scope without misleading disabled states or ambiguous input labeling.
