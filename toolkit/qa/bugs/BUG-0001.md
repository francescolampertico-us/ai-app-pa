# BUG-0001

## Bug ID
`BUG-0001`

## Tool
`background_memo_generator`

## Title
Background Memo injects an unrequested `FARA Registration` section into the memo body

## Severity
`medium`

## Found In Run
`2026-04-13 / BM-SMK-01 / Francesco + Codex review`

## Environment
- Branch: not captured
- Frontend: web app `/background-memo`
- Backend: FastAPI job flow
- Model: not captured

## Exact Repro Steps
1. Open the Background Memo page.
2. Enter the `BM-SMK-01` canonical input from `toolkit/qa/test_cases/background_memo.md`.
3. Keep the requested sections limited to:
   - Overview of Activities
   - Key Leadership
   - U.S. and NATO Relations
   - Funding and Membership
   - Relevant Publications
4. Run the tool.
5. Review the generated memo body and overview text.

## Exact Pasted Input
- Subject: `Jagello 2000`
- Memo Date: `April 13, 2026`
- Sections:

```text
Overview of Activities
Key Leadership
U.S. and NATO Relations
Funding and Membership
Relevant Publications
```

- Additional Context:

```text
Czech defense think tank founded by Zbyněk Pavlačík. Focus on transatlantic security cooperation. Hosts annual Prague NATO Days event, the largest public NATO airshow in the world.
```

- Disclosure entity override: `Jagello 2000`
- Disclosure sources: `lda`, `fara`

## Observed Behavior
The generated memo added a separate `FARA Registration` section that the user did not request. The overview also announced `fara registration` as part of the section sequence. This makes the memo feel overpacked and weakens the clean user-directed structure.

## Expected Behavior
The memo body should contain only the sections the user explicitly requested, plus the fixed sections defined by the output contract. Disclosure research can inform the content, but it should not silently create new top-level sections unless the user asks for them or the UI clearly offers that behavior.

## Why This Matters
This tool is supposed to respect user-defined section structure. Adding extra sections breaks trust in the prompt contract, makes outputs less predictable, and contributes to the readability problem the user noticed.

## Evidence / Output Excerpt
Overview excerpt:

```text
It begins with fast facts, followed by detailed sections on overview of activities, key leadership, u.s. and nato relations, funding and membership, relevant publications, fara registration.
```

Unexpected section heading:

```text
FARA Registration
FARA registration documents indicate that Jagello 2000 maintains an active foreign agent relationship within the United States.
```

## Suspected Layer
`backend`

## Claude Fix Request
Update the Background Memo generation flow so disclosure research can enrich requested sections without auto-inserting a new top-level section that the user did not request. Preserve the exact user-requested section list in the memo body and keep the overview aligned with that same section set.
