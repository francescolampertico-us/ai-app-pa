# BUG-0006

## Bug ID
`BUG-0006`

## Tool
`media_list_builder`

## Title
Media List Builder can complete successfully without showing the generated contact list

## Severity
`high`

## Found In Run
`2026-04-13 / MLB-SMK-01 / Francesco + Codex review`

## Environment
- Branch: not captured
- Frontend: web app `/media-list`
- Backend: FastAPI job flow
- Model: not captured

## Exact Repro Steps
1. Open the Media List Builder page.
2. Run the `MLB-SMK-01` canonical input from `toolkit/qa/test_cases/media_list_builder.md`.
3. Wait for the job to reach completed status.
4. Check whether contacts are rendered in the page.

## Exact Pasted Input
- Policy issue: `AI safety regulation and mandatory pre-deployment testing requirements`
- Geographic scope: `National (US)`
- Number of contacts: `20`
- Media types: `mainstream, print, broadcast, digital, trade`

## Observed Behavior
The tool reported that the media list was generated, but no visible results appeared in the UI.

## Expected Behavior
When the job completes successfully and the backend returns contacts, the page should render the generated contact list, summary metrics, and result controls.

## Why This Matters
This is a core success-path failure. A successful generation is not useful if the user cannot see or act on the list.

## Evidence / Output Excerpt
User report:

```text
It says the media list was generated but I don't see anything
```

Code context:

```text
The results panel is gated on `job?.status === 'completed' && contacts.length > 0`, so any mismatch between backend result shape and frontend `contacts` mapping will produce a silent completed-with-no-results state.
```

## Suspected Layer
`frontend`

## Claude Fix Request
Fix the Media List Builder success path so a completed job with returned contacts always renders visible results in the UI. If the backend returns zero contacts, show an explicit empty state instead of a silent completed state.
