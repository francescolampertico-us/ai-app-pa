# BUG-0002

## Bug ID
`BUG-0002`

## Tool
`legislative_tracker`

## Title
Legislative Tracker summary job completes but no summary is shown in the UI

## Severity
`high`

## Found In Run
`2026-04-13 / LT-SMK-01 / Francesco + Codex review`

## Environment
- Branch: not captured
- Frontend: web app `/legislative`
- Backend: FastAPI job flow
- Model: not captured

## Exact Repro Steps
1. Open the Legislative Tracker page.
2. Run the `LT-SMK-01` canonical input from `toolkit/qa/test_cases/legislative_tracker.md`.
3. Confirm search results load and add a plausible bill to the watchlist.
4. Run the summarize action for the bill.
5. Observe that the job reaches completed state but no summary appears in the UI.

## Exact Pasted Input
- Search mode: `Broad Search`
- Keywords / Topic: `artificial intelligence`
- Jurisdiction: `US (Federal)`
- Year: `2026`
- Results: `Top 10`

## Observed Behavior
Search results and watchlist behavior worked, but the summarize step did not present visible output to the user even though the job reported completion.

## Expected Behavior
When a summarize job completes, the page should show the generated summary and expose the summary download controls.

## Why This Matters
This breaks one of the core tool actions. A successful backend summary is not useful if the UI does not render it, and users will interpret the tool as broken even when the job status says completed.

## Evidence / Output Excerpt
User report:

```text
Results good, watchlist good, problem with summary. It says completed but no output is shown.
```

Likely frontend logic gap from code review:

```text
The page renders the summary only inside the selected-bill panel, but summarize actions triggered from the watchlist path do not appear to set selectedBill before showing the result.
```

## Suspected Layer
`frontend`

## Claude Fix Request
Fix the Legislative Tracker UI so any completed summarize action produces visible summary output. In particular, make sure summarize-from-watchlist and summarize-from-search both set the state needed to render the summary panel and related download controls.
